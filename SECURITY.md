# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security problems.

- Preferred: use GitHub's private vulnerability reporting for this repository.
- Include: impact, reproduction steps, affected paths, and any proof-of-concept details.

## Response Expectations

- Initial acknowledgment target: within 72 hours.
- Triage and severity assessment: as quickly as possible based on impact.
- Fix and disclosure timing: coordinated after mitigation is available.

## Scope

This project includes automation scripts, Ansible content, and GitHub workflows.
Potentially sensitive items include credentials, tokens, key material, and backup or migration payload handling.

## Disclosure

After remediation, a summary may be published in release notes or repository discussions.
